# KOSPI AI 모니터링 시스템 스냅샷 및 작업 이력

본 파일은 수일 간 진행된 KOSPI 종목 추천 및 모니터링 시스템 작업의 컨텍스트를 보존하고 나중에 이어서 작업하기 위해 기록되었습니다.

## 1. 현재 발생한 텔레그램 메시지 이슈 분석
- **현상**: 텔레그램에 AI의 화려한 설명 대신 "🤖 자동 기술적 분석 리포트 (데이터 원본 전문)" 이라는 딱딱한 원본 데이터만 전송됨.
- **원인**: 시스템을 구현하고 테스트하는 과정에서 짧은 시간에 액션을 수십 번 가동함. 이로 인해 구글 AI 스튜디오 API의 **무료 할당량(일일 Quota)** 이 초과됨 (429 Quota Exceeded 에러 발생).
- **결론**: 에러가 아니며 코드는 완벽히 작동 중. 한국 시간 자정이 지나 무료 한도가 갱신되면 다시 AI가 작동하여 사람의 언어로 리포팅됨.

## 2. 전체 시스템 아키텍처 및 로직
- **100% 무료 서버리스 환경**: 텔레그램 챗봇 방식은 서버비(호스팅)가 발생하므로, GitHub Actions를 이용해 예약된 시간에 딱 한 번씩만 스크립트를 돌려 무료로 유지되는 아키텍처를 채택함.
- **주말/공휴일 휴장일 스킵 (holidays 라이브러리)**: API 할당량과 GitHub Actions 낭비를 막기 위해 파이썬 `holidays`를 도입함. 주말이거나 한국 공휴일에는 스크립트가 실행되어도 데이터를 부르지 않고 바로 종료됨. (예외: 테스트를 위해 GitHub 웹 콘솔이나 로컬 터미널에서 `workflow_dispatch` 수동 실행을 강제로 누를 경우에는 예외적으로 무조건 작동함)
- **원본 텍스트 투척(백업) 통신망**: 오늘처럼 구글 서버가 한도 초과(429)로 차단되거나 오류가 나면 묵묵부답이 되지 않고, 분석된 "숫자 원본"이라도 텔레그램으로 확실하게 보내주도록 Fallback 로직 완성.

## 3. GitHub Actions 스케줄러 세팅
- **종합 매수 추천 전종목 분석 (`analyze.yml`)**: 
  - KST 기준 매일 아침 **07:50**에 강제 분석 시작. (수많은 코스피 종목을 분석하므로 약 40분 뒤인 **08:30** 전후 텔레그램 도착)
- **실시간 감시 및 에프터마켓 점검 (`monitor.yml`)**: 
  - KST 기준 평일 장 시작 시간인 **09:00부터 18:00** (시간외 단일가 마감까지 포함) 동안 30분 마다 스캔 및 푸시 메시지 발송.

## 4. 내일 테스트를 위한 액션 (Next Steps)
- 코스피 장이 열리거나 (또는 주말이라도 강제 수동 실행으로) 테스트해보고 싶다면, 터미널에서 다음 명령어로 즉시 가동.
  ```bash
  ./gh workflow run monitor.yml
  ./gh workflow run analyze.yml
  ```
- 구글 무료 할당량이 리필되었다면, 기존처럼 "왜 팔아야 하는지" 자세한 해설이 리포팅됨.
